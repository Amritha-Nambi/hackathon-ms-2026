# UI Tone & Writing Guide

## Tone
- Tone: Professional, trustworthy, efficient, precise, and helpful. The app voice should feel like a skilled sales operations partner delivering clear, action-ready summaries.

## UI copy
- Title: Sales Call Summary Generator
- Tagline: Turn transcripts into CRM-ready summaries in seconds.
- Section headings:
  - Transcript Input
  - Summary Output
  - Key Discussion Points
  - Customer Needs
  - Next Actions
- Button labels:
  - Upload transcript
  - Paste transcript
  - Generate summary
  - Reset input
- Helper text:
  - Paste the call transcript or upload a text file to create a concise summary.
  - The summary will include key discussion points, customer needs, and next actions.
  - Keep the transcript focused on the call conversation for best results.
- Empty state / upload prompt:
  - No transcript yet. Paste the call text or upload a file to begin.
  - Ready to summarize your call. Add the transcript and click Generate summary.

## Guidelines
- Use short, direct phrases.
- Keep labels and headings crisp and task-oriented.
- Avoid hype and unnecessary adjectives.
- Use plain business language that sales reps and account managers understand.
- Prioritize clarity over cleverness.
- Make actions obvious: the user should know what to do next.
- Keep helper text supportive and factual.
- Use consistent terminology: transcript, summary, key discussion points, customer needs, next actions.
- If metadata is optional, explain that clearly without overloading the user.
- Ensure the interface feels reliable and efficient, not flashy.

## Color palette
- Primary: `#0078D4` (Microsoft blue)
- Accent: `#005A9E` (strong action color)
- Background: `#F3F2F1` (neutral light)
- Surface: `#FFFFFF` (clean panel background)
- Text primary: `#201F1E` (dark gray for readability)
- Text secondary: `#605E5C` (muted gray for helper text)
- Success: `#107C10` (positive action/confirmation)
- Border / divider: `#C8C6C4` (subtle neutral divider)
